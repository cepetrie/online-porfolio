  • Website: Washington Junior Golf Association
  • Service: Designed and developed a responsive custom WordPress theme for Washington Junior Golf Association, a youth program designed to develop golfing skills.

Project lead for a team of two designers and three developers, including myself. Our client for this project was a non-profit organization, the Washington Junior Golf Assocation.

As Project Lead I handled all communication with the client and was responsible for making sure that our deliverables were complete and on time.

One of the challenges of this website was displaying the sub-navigation in the sidebar when the user is on a page that has sub-navigation, even when they are on a sub-page. I had to use the wp_list_pages() function to get the child pages and the get_the_title() function to get the parent page title, a method that my teacher Mike Sinkula showed me.

When it came time to migrate our locally developed site to the client's sever we used the WP Migrate DB plugin, which I loved. It exports your database as a MySQL data dump (much like phpMyAdmin), does a find and replace on URLs and file paths, then allows you to save it to your computer.

An Inside Look

BEFORE:

AFTER:


  • Website: An Inside Look
  • Service: Designed and developed a responsive website using HTML5, CSS3, JavaScript and jQuery for my client, Kristen Scott, an interior designer.

I chose Blueberry, a jQuery image slider plugin, for the front page slideshow. The main reason I chose it was because it is it is responsive and automatically resizes images according to the size of the window. It was very simple to implement, configure and use. The only restriction was that all images had to be the same size in the image slider.

The biggest challenge for me was how to showcase my client's portfolio, the most important aspect of an interior designer's website. I ended up settling with FancyBox, a tool for displaying images in a "lightbox" that floats overtop of a web page. It is built using jQuery. I created separate galleries for each of the four residences and chose to display each gallery with only one preview picture, so that the fancyBox has to be opened manually.

For the contact form I used Clearbox, which is a professional image and content viewer overlay window written in JavaScript. Although different types of contents are supported by this plugin, I only used HTML.

The navigation is displayed as a dropdown menu on smaller viewports. I used the Mobile Dropdown Conversion created by Astuteo to make this happen. It is powered by CSS media queries and a little jQuery. It's single base color was easy to adjust. I added the mobile navigation to my media queries, at the point when the browser window has a max-width of 700px.

Node.js blog application


  • Website: Node.js blog application
  • Service: Created a Node.js blog content management system using a Hapi.js web framework.

For this project I created a simple Node.js blog content management system, using Hapi.js on the server. Visitors are able to view a list of posts on the site in reverse-chronological order, with "read more" links. A single post view is available for viewing the post in full.

The data for this application is stored in a SQLite database, which was installed using the sqlite3 module. This allowed me to create a small, local databse, without having to run a separate MySQL server. It uses SQL operations to create the tables, and insert, update and delete records based on user action.

Users can view a list of posts, but post creation and editing is restricted to authenticated users only. When the user logs in, the user's ID and password is stored in a cookie and validated against the user table. If the login is valid, the member-only page can load. If not, the user is returned to the Login page. The Crypto module was incorporated to calculate a hash.

Handlebars/Mustache was used for dynamic templating and to output data.

My application uses an MVC architecture and is broken down into three main parts: the model, the view and the controller. The handlers for each route are my controllers, because they accept data from the browser and send data to the view.
